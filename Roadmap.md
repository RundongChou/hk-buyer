# 香港买手跨境代购 Roadmap（10 Sprints）

## 0. 北极星基准（原始产品说明文档）

以下内容完整作为本项目的北极星约束与需求上限，后续所有 Sprint 需求若与本章冲突，以本章为准。

### 0.1 产品定位
1. 面向内地用户，提供香港正品商品（药房、药妆、零售）的一站式下单与跨境履约服务。
2. 平台核心价值：正品保障、价格优势、可追踪履约、买手激励撮合。
3. 标准履约承诺：用户下单后 7-15 天签收（按合规清关路径）。

### 0.2 北极星目标与指标
1. NSM：30天跨境履约闭环订单数（CCO-30）。
定义：支付成功 + 买手接单 + 采购完成 + 清关出库 + 用户签收。

2. 护栏指标：

| 指标 | 目标 |
| --- | --- |
| 假货投诉率 | <= 0.3% |
| 超时未接单率（72h） | <= 15% |
| 订单取消率 | <= 8% |
| 7-15天履约达成率 | >= 90% |
| 合规清关成功率 | >= 99.5% |
| 用户NPS | >= 45 |

### 0.3 用户与角色
1. C端用户：浏览、下单、支付、追踪、售后。
2. 香港买手：接单、采购、上传凭证、交仓、结算。
3. 平台运营：选品、定价、活动、异常处理。
4. 平台风控：资质审核、交易风控、反作弊、合规审计。
5. 仓配与物流：入仓、分拣、清关、末端派送。
6. 财务：账务、分账、结算、发票与对账。

### 0.4 业务闭环
1. 用户选品下单支付。
2. 系统生成采购任务并在买手池发布。
3. 买手接单并采购，提交小票/凭证。
4. 商品交付香港集货仓，质检与入库。
5. 合规清关后发往内地。
6. 用户签收，平台分账结算。
7. 异常订单走售后与仲裁。

### 0.5 核心功能清单

#### 0.5.1 前台（用户端）
1. FR-U01 账号体系：手机号注册登录、实名认证、收货地址管理。
2. FR-U02 商品浏览：类目、品牌、关键词、筛选（价格/功效/库存）。
3. FR-U03 SKU详情：图文详情、规格、港币/人民币价格、预计时效、库存状态、正品说明。
4. FR-U04 购物车与下单：凑单、优惠券、运费估算、税费提示。
5. FR-U05 支付：微信/支付宝，支付状态回调与失败补偿。
6. FR-U06 订单中心：待接单/采购中/清关中/运输中/已签收全链路追踪。
7. FR-U07 售后中心：退货退款、少件破损、真伪争议。
8. FR-U08 消息通知：接单、发货、异常、退款、活动通知。
9. FR-U09 评价体系：商品评价、买手服务评分、平台服务评分。
10. FR-U10 推荐与搜索优化：热销、复购推荐、相似替代商品。

#### 0.5.2 前台（买手端）
1. FR-B01 入驻与审核：身份、资质、支付账户、信用评估。
2. FR-B02 任务大厅：按品类/利润/区域筛选任务，抢单与放弃。
3. FR-B03 采购执行：采购清单、门店建议、提货码、交仓码。
4. FR-B04 凭证上传：小票、发票、商品批次、有效期、实拍图。
5. FR-B05 履约管理：超时预警、缺货上报、替代建议。
6. FR-B06 收益中心：订单收益、服务费、结算周期、提现。
7. FR-B07 信誉体系：接单率、超时率、投诉率、信用分。

#### 0.5.3 管理后台
1. FR-A01 商品中心：SPU/SKU、图文、属性、禁售规则、上/下架。
2. FR-A02 价格中心：基础价、服务费率、动态加价策略、价格保护。
3. FR-A03 库存中心：可售库存、在途库存、缺货预警、锁库存。
4. FR-A04 订单调度：自动派单、人工干预、任务重派、合单拆单。
5. FR-A05 买手管理：准入审核、分层、黑白名单、处罚与激励。
6. FR-A06 仓配清关：入仓、质检、打包、关务资料、运单轨迹。
7. FR-A07 风控合规：反欺诈、真伪抽检、敏感品类拦截、审计日志。
8. FR-A08 客服工单：退款、纠纷、补偿、仲裁结论。
9. FR-A09 财务结算：分账、对账、手续费、技术服务费、税务报表。
10. FR-A10 数据看板：转化漏斗、履约时效、毛利、复购、异常预警。

### 0.6 核心规则（关键）
1. 无人接单提价规则：任务发布后 72 小时无人接单，自动提价 `max(5%, 20元)`，每 24 小时最多一次，单订单最多提价 3 次，封顶 20%。
2. 缺货处理规则：买手上报缺货后可触发替代推荐；用户确认替代或部分退款。
3. 正品保障规则：凭证必传 + 仓端抽检 + 高风险品类加严审查。
4. 分账规则：用户支付金额按“商品成本、买手收益、物流成本、平台手续费/技术服务费”自动拆分。
5. SLA规则：接单 SLA 72 小时；全链路履约目标 7-15 天。

## 1. 北极星指标与商业化指标（重构版）

### 1.1 North Star Metric（NSM）
**30天跨境履约闭环订单数（CCO-30）**

闭环定义（同一用户 30 天内，至少 1 单满足）：
1. 支付成功；
2. 买手接单并上传采购凭证；
3. 仓配入库与清关出库；
4. 用户签收完成。

### 1.2 护栏指标（防偏航）
1. 假货投诉率 <= 0.3%。
2. 超时未接单率（72h） <= 15%。
3. 订单取消率 <= 8%。
4. 7-15天履约达成率 >= 90%。
5. 合规清关成功率 >= 99.5%。
6. 用户 NPS >= 45。

### 1.3 商业化核心指标
1. 订单毛利率（含技术服务费）>= 目标阈值。
2. 买手侧有效接单率与活跃买手数持续增长。
3. 30 天复购率持续提升。
4. 风险成本（赔付/拒付/假货争议）占比持续下降。

### 1.4 价值主张（从可商业化角度）
1. 用户侧：香港正品 + 价格优势 + 全链路可追踪，降低跨境购买决策成本。
2. 买手侧：透明利润与阶梯激励，提升接单意愿与履约稳定性。
3. 平台侧：撮合手续费 + 技术服务费 + 运营工具沉淀，形成可规模化模型。

## 2. 竞品研究结论（2026-03-14）

> 说明：以下基于公开检索可得信息（中文搜索、App Store、微信生态公开资料）整理，用于 Sprint 1 的功能取舍，不作为法律或财务结论。

### 2.1 竞品样本（网站/App/小程序）
1. **Buy&Ship（国际代购/转运）**
   - 公开强调“海外代购 + 集运 + 关税与物流规则透明”。
   - 启发：需要把“费用构成与运输规则”前置到用户下单决策。
2. **HKTVmall（香港电商平台）**
   - 公开强调“海量SKU、优惠活动、履约体验和会员权益”。
   - 启发：需要建立可持续的品类深度和运营活动系统，而非一次性代购。
3. **屈臣氏官方微信小程序（中国内地）**
   - 公开信息显示其小程序承载“新品首发、会员、优惠券、便捷下单”。
   - 启发：小程序端要优先保留高频交易能力和会员触达能力。
4. **持牌药房代购 App（如药房聚合类）**
   - 公开强调“药房品类覆盖、商品搜索、订单追踪”。
   - 启发：药妆/药品类目必须强化资质信息、适用说明、售后规则提示。
5. **京东国际（跨境电商）**
   - 公开规则强调“跨境商品正品保障、履约时效与售后规则明确”。
   - 启发：平台需要可审计的正品链路与明确售后边界。

### 2.2 竞品共性能力
1. 商品供给密度高（SKU 多、品牌全、活动频次高）。
2. 支付与履约状态展示清晰。
3. 会员与促销体系完整（券、满减、积分、专享价）。

### 2.3 当前缺口与可突破点
1. **可追溯不完整**：多数“代购”场景缺少凭证结构化与证据链查询。
2. **供给激励不系统**：买手接单靠人工经验，缺少自动加价与信誉协同。
3. **异常闭环不彻底**：缺货替代、部分退款、争议仲裁链路割裂。
4. **平台化弱**：缺少从买手、仓配、关务到财务的一体化后台。

### 2.4 Sprint 1 功能吸收与取舍策略
1. 吸收：搜索筛选、SKU详情、优惠触达、订单可视化、买手任务大厅。
2. 强化：采购凭证结构化、正品保障证据链、无人接单动态激励。
3. 延后：复杂会员分层、营销自动化、多渠道投放归因（放到 Sprint 7/9）。

## 3. 核心卖点（商业化导向）

1. **正品可证据化**：用户不仅看“正品承诺”，还能看采购凭证、抽检记录、批次信息。
2. **价格可解释**：订单展示“商品价 + 履约服务费 + 物流税费估算”，让价格优势透明可比。
3. **供给可激励**：无人接单自动提价 + 买手信誉分联动，提升接单率并控制履约时效。
4. **履约可追踪**：从接单、采购、入仓、清关到签收全链路状态可视化。
5. **平台可盈利**：撮合手续费 + 技术服务费 + 供应链协同效率提升，形成长期经营模型。

## 5. Sprint 1 PRD（基石 Sprint，详细版）

### 5.1 Sprint 名称
`Sprint 1：香港买手跨境代购最小交易闭环 MVP（竞品关键能力落地）`

### 5.2 Sprint 目标（必须达成）
1. 在 1 个 Sprint 内打通最小交易闭环：用户下单支付 -> 买手接单采购 -> 上传凭证 -> 订单状态可追踪。
2. 建立“正品可追溯”最小能力：每单至少 1 条采购凭证，可在后台审计。
3. 跑通任务发布与接单基础规则，支持 72h 未接单预警（自动提价逻辑可先 V0 上线）。
4. 形成首版事件埋点，能够计算 CCO-30 漏斗的前两段（支付、接单）。

### 5.3 竞品映射后的产品策略

#### 5.3.1 必须吸收的能力（Sprint 1 立刻做）
1. 商品可搜索、可筛选、可查看详情（对标 HK 电商体验）。
2. 下单支付体验完整且状态明确（对标跨境电商的交易稳定性）。
3. 买手任务大厅与抢单机制（对标代购平台供给组织方式）。
4. 消费者可见的履约状态节点（对标主流电商履约可视化）。

#### 5.3.2 必须差异化的能力（Sprint 1 就做）
1. 采购凭证结构化（小票、批次、有效期、实拍）而非仅图片上传。
2. 订单级“正品保障页”原型：展示凭证摘要、抽检状态、免责声明。
3. 无人接单预警 + 提价建议看板（运营可手动确认执行）。

#### 5.3.3 延后能力（避免 Sprint 1 过载）
1. 会员体系、积分、复杂促销规则。
2. 自动清结算与税务报表。
3. 多渠道（App/小程序）统一运营后台。

### 5.4 用户与核心场景

用户角色：
1. 内地用户（下单方）
2. 香港买手（履约方）
3. 运营人员（调度方）

核心场景：
1. 用户从 SKU 列表选择商品，完成下单支付并查看“待买手接单”。
2. 买手在任务大厅接单，采购后提交凭证并等待运营审核。
3. 运营审核凭证通过后，订单进入“待入仓/运输中”状态，用户可追踪。

### 5.5 范围（In Scope / Out of Scope）

In Scope：
1. 用户端：注册登录、商品浏览、购物车、下单支付、订单追踪。
2. 买手端：入驻申请、任务大厅、接单、凭证上传、收益预估。
3. 后台：商品管理、订单调度、凭证审核、异常标记。
4. 风控：基础黑名单、敏感品类拦截、审计日志。

Out of Scope：

Roadmap Sprint 1 目标是“竞品对标最小交易闭环 MVP”。当前仓库仅有文档，尚未形成可运行的前后端分离工程，也没有可验证的下单-接单-凭证-审核-状态回传链路。

## Roadmap Sprint 5 PRD（2026-03-15）

### 1. 背景与问题定义
Roadmap 总览中 Sprint 5 的既定目标是“动态加价与重派引擎”：`72h 无人接单提价 -> 自动重派 -> 接单或终止`。  
当前系统已在 Sprint 4 实现买手分层与准入，但对于长期无人接单任务仍停留在“可统计、不可自动处理”阶段，导致超时任务持续堆积，影响 CCO-30 的履约入口与运营效率。

### 2. Sprint 目标（可验证）
1. 上线 72h 超时未接单任务自动提价执行器，可批量处理 `PUBLISHED` 且已超时任务。
2. 落地 24h 频控与最多 3 次提价规则，且单订单累计提价不超过订单金额 20%。
3. 实现自动重派与失败终止：提价后重置接单窗口；达到提价上限后自动终止并记录原因。
4. 在管理后台与数据平台提供可见的执行结果与核心指标（提价覆盖、重派次数、终止数）。

### 3. 非目标（本 Sprint 不做）
1. 不做仓配入仓、清关、物流链路改造（Roadmap Sprint 6 范围）。
2. 不做复杂机器学习定价，仅执行固定规则 `max(5%, 20元)` + 封顶策略。
3. 不做买手结算与平台分账（Roadmap Sprint 8 范围）。
4. 不做任何绕关/走私能力，严格沿用合规清关路径。

### 4. 用户故事与使用场景
1. 管理后台：作为运营，我可以触发并查看“自动提价执行结果”，知道哪些任务被重派、哪些任务被终止。
2. 买手端：作为买手，我在任务大厅可看到已提价重派后的任务与最新建议加价金额。
3. 数据平台：作为运营分析，我可查看自动提价覆盖率、提价后接单转化、频控跳过数与终止数。

### 5. 范围（In Scope / Out of Scope）
In Scope：
1. 超时任务扫描、提价计算、24h 频控、最多 3 次提价、20% 封顶。

3. 达到上限后的自动终止与终止原因留痕。

Out of Scope：

   回滚：推荐失败时回退到已上架热销 SKU 列表，不影响下单主链路。
2. 风险：活动重复触达导致用户骚扰。
   回滚：通过 `campaign_id + user_id` 唯一约束去重，重复发布自动幂等忽略。
3. 风险：券归因字段为空导致增长指标偏差。
   回滚：下单归因写入失败不阻塞下单，但记录日志并在指标端单独统计“未归因订单”。
4. 风险：本机无 JDK8/Maven，后端自动化测试无法执行。
   回滚/缓解：在 CI 或具备环境的开发机补跑后端测试。

### 12. 与北极星指标映射（本 Sprint 如何贡献 CCO-30 与护栏）
NSM（CCO-30）贡献：
1. 通过会员分层、活动触达与复购推荐提升 30 天内回流下单，直接扩大跨境履约闭环订单池。

护栏映射：
1. 假货投诉率：不修改正品与仲裁流程，沿用 Sprint 7 风控闭环。
2. 72h 超时未接单率：不修改派单引擎，沿用 Sprint 5 机制；复购增加后需持续监控供给压力。
3. 订单取消率：通过精准券与推荐减少冲动下单后取消风险，并跟踪 `order_cancel_rate`。
4. 7-15天履约达成率：不改仓配清关链路，保持 Sprint 6 护栏。
5. 合规清关成功率：不改清关路径，仅做增长侧触达与推荐。
6. 用户NPS：会员权益透明、推荐更相关、券包可见性提升，有助于提升服务感知。

## Roadmap Sprint 10 PRD（2026-03-15）

### 1. 背景与问题定义
Roadmap Sprint 10 目标是“稳态运营与优化 V1（A/B、监控告警、成本与时效优化）”。
当前项目已在 Sprint 1-9 打通交易、履约、售后、结算、增长闭环，但仍缺少持续优化机制：
1. 派单策略缺少可追踪实验分流，无法量化“策略是否真的提升 7-15 天履约达成率且不牺牲成本”。
2. 护栏指标虽可查询，但缺少“自动阈值判定 + 告警事件”闭环，运营发现问题依赖人工巡检。
3. 缺少“实验结果 -> 策略建议”的统一输出，不利于规模化迭代与稳定增长。

### 2. Sprint 目标（可验证）
1. 交付 A/B 实验最小能力：后台可创建并激活实验，支付后任务派发可按实验分流并留痕。
2. 交付全链路护栏告警最小能力：支持一键评估并生成/恢复告警（72h超时未接单率、订单取消率、7-15天履约达成率、合规清关成功率、假货投诉率）。
3. 交付优化指标面板：输出实验分组时效与成本指标、策略建议，支撑“上线 -> 监控 -> 复盘 -> 优化”闭环。
4. 全部改动保持 `TypeScript + TSX / Java 8 + Spring MVC / MySQL` 技术栈一致，且仅沿合规清关路径实现。

### 3. 非目标（明确不做）
1. 不做外部 A/B 平台接入（如三方实验平台），仅做站内实验与分流。
2. 不做实时流式告警系统（短信/电话/IM）与多渠道通知编排，本 Sprint 仅做平台内告警事件。
3. 不做复杂机器学习自动调参，仅提供规则型策略建议。
4. 不实现任何绕关/走私相关能力，继续严格遵守合规清关路径。

### 4. 用户故事与使用场景
1. 管理后台（运营/策略）：作为运营，我希望创建并激活一个派单优化实验，观察 Control/Treatment 的时效与成本差异。
2. 管理后台（风控）：作为运营，我希望一键评估护栏指标并看到当前 OPEN 告警列表，快速定位风险。
3. 数据平台：作为数据分析，我希望查看实验分流规模、分组履约达成率、分组成本率、策略建议。

### 5. 范围（In Scope / Out of Scope）
In Scope：
1. 实验配置与激活（单活实验）能力。
2. 订单支付后任务发布分流：按实验配置决定分组并记录策略参数（加价、SLA）。
